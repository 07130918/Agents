# 攻撃面チェックリスト (6 カテゴリ)

Phase 2 探索および Phase 5 新規発見時の網羅チェックリスト。

## 1. 認証・セッション管理

- OAuth / OIDC フロー実装
- セッション生成・検証ロジック
- JWT 署名・検証 (アルゴリズム、秘密鍵管理、有効期限)
- パスワードハッシュ (使用している場合)
- Cookie 設定 (`httpOnly`, `secure`, `sameSite`, `domain`, `path`)
- セッション無効化 (ログアウト、ブラックリスト)

## 2. 認可・アクセス制御

- ロールベースアクセス制御 (RBAC) の実装
- API エンドポイントの保護状態 (`public` vs `protected` vs `admin`)
- IDOR (安全でない直接オブジェクト参照)
- 水平・垂直権限昇格の可能性
- 管理者昇格ロジック

## 3. CSRF・トランスポートセキュリティ

- CSRF トークンの実装有無
- SameSite Cookie 設定
- CORS 設定
- HTTPS 強制 (HSTS)
- State パラメータの署名検証 (OAuth)

## 4. 入力検証・インジェクション

- SQL インジェクション (ORM バイパス、raw query)
- XSS (テンプレート、マークダウン、`dangerouslySetInnerHTML`)
- コマンドインジェクション
- パストラバーサル (`../`, null byte)
- 入力長制限 (Zod `.max()`、バリデーション)
- ファイルアップロード検証

## 5. インフラ・サーバー設定

- セキュリティヘッダー (helmet, CSP, X-Frame-Options 等)
- レートリミット
- ボディサイズ制限
- trust proxy 設定
- デバッグモード・開発者向けエンドポイント
- 環境変数のデフォルト値 (空文字列フォールバック)

## 6. ログ・監視・情報漏洩

- エラーメッセージの情報漏洩 (スタックトレース、DB 情報)
- デバッグログの本番残存
- API キーのクライアント露出
- localStorage / sessionStorage の機密データ保存
- ルート情報の露出
- `.env` ファイルの管理状態

## Phase 5 新規発見の追加チェック

- DB マイグレーション履歴の整合性 (UNIQUE 制約回帰、PK 喪失)
- ボディパーサーのサイズ制限 (DoS)
- localStorage / sessionStorage の機密データ
- trust proxy 未設定による IP 偽装
- デバッグ用プラグイン / ミドルウェアの本番残存
- ORM スキーマと DB の乖離
- 依存パッケージのパッチファイルによるセキュリティ変更
- 開発サーバーのネットワーク公開設定 (`host: true`)
- トランザクション未使用の複数 DB 書き込み (TOCTOU)
- 環境変数の空文字列デフォルト
