# スコアリング & レポートテンプレート

Phase 6 スコア計算および最終レポート作成時に参照。

## 6 カテゴリ重み付きスコア

| カテゴリ | 重み | 評価対象 |
|---|---|---|
| 認証・セッション管理 | 25% | OAuth, JWT, Cookie, セッション |
| 認可・アクセス制御 | 20% | RBAC, IDOR, 権限昇格 |
| CSRF・トランスポート | 15% | CSRF, CORS, HTTPS, State |
| 入力検証 | 20% | SQLi, XSS, Path Traversal, Validation |
| インフラ・サーバー設定 | 10% | Headers, Rate Limit, Body Size |
| ログ・監視・情報漏洩 | 10% | Debug Log, API Key Exposure, Error Info |

## 深刻度別減点 (各カテゴリ 100 点満点から減点)

| 深刻度 | 減点 | 定義 |
|---|---|---|
| Critical | -30 | リモートから認証不要で悪用可能、データ漏洩/完全制御 |
| High | -15 | 認証済みユーザーが権限昇格可能、重要データへのアクセス |
| Medium | -8 | 限定的な情報漏洩、特定条件下で悪用可能 |
| Low | -4 | 情報開示、ベストプラクティス違反、防御策の不足 |

## スコア計算式

```
各カテゴリスコア = max(0, 100 - Σ(該当する脆弱性の減点))
総合スコア = Σ(各カテゴリスコア × 重み) → 小数点以下四捨五入
```

## スコアの解釈

| スコア | 評価 |
|---|---|
| 80-100 | 良好 — 軽微な改善のみ必要 |
| 60-79 | 要改善 — 計画的な対応が必要 |
| 40-59 | 危険 — 早急な対応が必要 |
| 0-39 | 緊急 — 即座の対応が必要、本番運用不可 |
