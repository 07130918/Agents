# /codex-review (/cr)

現在のブランチの差分を Codex MCP に渡してレビューする。エイリアス: `/cr`。

## 手順

1. ベースブランチを判定 (`develop` を優先、無ければ `main`)
2. `git diff <base>...HEAD --no-color` で差分取得
3. 差分が空なら中止、50,000 文字超なら警告
4. `mcp__codex__codex` に下記プロンプト + 差分を渡す
5. 結果を日本語で表示

## レビュープロンプト

Codex MCP に渡す本文:

```
あなたは経験豊富なGoogle出身のシニアソフトウェアエンジニアです。
以下のgit diffを詳細にレビューし、日本語で包括的なコードレビューを提供してください。

レビュー観点:
1. バグの可能性: 潜在的なバグ、論理エラー、例外処理の不備
2. パフォーマンス: 性能上の問題、最適化の機会
3. 保守性: 可読性、構造、命名規則、重複コード
4. ベストプラクティス: 言語・フレームワーク固有の規約
5. セキュリティ: 脆弱性の可能性
6. 改善提案: 具体的で実装可能な改善案

出力形式:
- 各観点を明確に分けて記載
- 重要度を [高] / [中] / [低] のラベルで分類 (絵文字は使わない)
- ファイル名・行数を含めて指摘
- 改善案は実装例付き

以下がレビュー対象のdiffです:
```
