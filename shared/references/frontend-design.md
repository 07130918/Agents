# frontend-design

この参照は複数 CLI から使う共通本体です。実行中の CLI に対応する節だけを使用してください。

## Codex / .agents 版

このスキルは、凡庸な「AIっぽい」美学を避けた、特徴的でプロダクション品質のフロントエンドを作り上げるためのガイドです。美的ディテールや創造的な選択を徹底的に重視しながら、実際に動作するコードを実装します。

ユーザーは、作成したいコンポーネント、ページ、アプリケーション、インターフェースなどのフロントエンド要件を提示します。目的、対象ユーザー、技術的制約などの文脈が含まれることもあります。

## デザイン思考

コーディングの前に文脈を理解し、大胆な美的方向性を決める：
- **目的**: このインターフェースはどんな問題を解決するのか？誰が使うのか？
- **トーン**: 極端を選ぶ。例：極限のミニマル、カオスなマキシマリズム、レトロフューチャー、有機的／自然、ラグジュアリー／洗練、遊び心／おもちゃ風、エディトリアル／雑誌、ブルータリズム／無骨、アールデコ／幾何学、柔らかいパステル、工業的／実用主義など。インスピレーションとして使いつつ、真に方向性に合ったデザインをする。
- **制約**: フレームワーク、パフォーマンス、アクセシビリティなどの技術要件。
- **差別化**: 何が「忘れられない」要素になるか？見た人が思い出す“一つの特徴”は何か？

**重要**: 明確なコンセプト方向を選び、精密に実行すること。大胆なマキシマリズムも、洗練されたミニマリズムも成立する──鍵は強度ではなく“意図性”。

そのうえで、以下を満たすコード(React)を実装する：
- プロダクション品質で機能的
- 視覚的に強烈で記憶に残る
- 一貫した美的視点
- 全てのディテールが緻密に洗練されている

## フロントエンド美学ガイドライン

重点ポイント：
- **タイポグラフィ**: 美しくユニークで興味深いフォントを選ぶ。Arial や Inter のような汎用フォントは避け、フロントエンドの美しさを高める独自性のあるフォントを使用。個性的なディスプレイフォントと洗練された本文フォントを組み合わせる。
- **色・テーマ**: 明確な美的方向性にコミットする。薄く散らばった配色よりも、強い主色＋鋭いアクセントが効果的。
- **モーション**: アニメーションやマイクロインタラクションを活用。Reactでは Motion ライブラリを使用。強いインパクトのある瞬間を重視：1回の美しいページロード演出(遅延アニメーションなど)は、散発的な小さな動きよりも大きな効果がある。スクロールやホバーで驚きを演出。
- **空間構成**: 予想外のレイアウト。非対称。オーバーラップ。斜めの流れ。グリッド崩し。広い余白 **または** 密度のコントロール。
- **背景・視覚ディテール**: 単色背景に逃げず、雰囲気・奥行きを作る。グラデーションメッシュ、ノイズ、幾何学パターン、レイヤー透明、劇的なシャドウ、装飾ボーダー、カスタムカーソル、粒子感オーバーレイなど、文脈に合った表現を加える。

決して使ってはいけないもの：
- Inter / Roboto / Arial / システム系の使い回されたフォント
- 白背景に紫グラデなどの使い古された配色
- ありきたりなレイアウト・コンポーネントパターン
- 文脈の個性がないテンプレ的デザイン

文脈に応じて創造的に解釈し、予想外の選択をする。デザインは常に異なるべき。ライト／ダークテーマ、フォント、全てを変える。世代ごとに“よくある選択”(例：Space Grotesk)に収束しないこと。

**重要**: 実装の複雑さは美的ビジョンと整合させる。
マキシマリズムには複雑なアニメーションやエフェクトが必須。
ミニマル／洗練系は抑制・精密さ・間の取り方・タイポグラフィの微調整が重要。
美しさはビジョンの「正確な実行」から生まれる。

Codex は卓越した創造性を持つ。妥協せず、枠を超え、本物のビジョンを持った作品を生み出すべきである。
